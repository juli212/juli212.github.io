Arrays and Hashes

February 14, 2016

What Are Arrays and Hashes??

Arrays and Hashes are both collections where date is stored, looked up, and modified. In the most current version of Ruby, both hashes and arrays are ordered which means the objects inside are held in a specific sequence and can be referenced by the number of their position. In previous versions of Ruby however, Arrays were ordered but hashes were not.

Any object including more arrays, hashes, clases, etc, can be stored in an array. Unlike arrays, hashes store objects in pairs, each of which consist of a key and a value. Values are retrieved from hashes through their corresponding keys. Values can be repeated throughout a hash but keys are unique and can only exist once per hash.

Sticky on the Codecademy Forum has a great explanation for the differences between arrays and hashes if there is more confusion!

Choosing: Array vs Hash!!

Functionally, arrays and hashes are very similar but accessing the data is different. In arrays, an object is accessed through its index number. Arrays start with position 0 so for example, if you want the 7th object in an array, you would actually return it with array(6). Objects in hashes are accessed through their keys which will then point at the value so if you want to call an object without knowing its numerical place, hashes are the way to go since you assign your own key: value pairs.

Here is an example of an array called winter_clothing followed by a hash that I will also call winter_clothing: 


			winter_clothing ["hat", "mittens", "boots", "pants", "socks", "sweater", "coat", "scarf" ]
		

			winter_clothing {	
						"hat"		=>	"head",
						"mittens"	=>	"hands",
						"boots"		=>	"feet",
						"pants"		=>	"legs",
						"socks"		=>	"feet"
						"sweater"	=>	"body"
						"coat"		=>	"body",
						"scarf"		=>	"neck" }

As you can see, the array is defined using [ ] while the hash is defined using { } but there are multiple ways to define both collections. In the hash above, the keys are on the left and the values are on the right. Two of the values are repeated but none of the keys can be. Below is the code to return the string "socks" from first the array and the value that socks points to from the hash ("feet"): 


			winter_clothing(4)
			>"socks"
		

			winter_clothing(socks)
			>"feet"

So if I just wanted to go through a list of clothing I should wear in the winter to make sure I remembered everything, I would use an array. On the other hand, if I had all of this winter clothing but did not know which part of my body to wear it on, I could use a hash. I would give my key which is the item of clothing and it would return the value which is the correct body part. In some cases (like for socks and boots), the value/body part is the same even though the keys are not. If the same key showed up twice, a hash would return the most recent value for that key as the key's value because remember, each key is unique within the hash!
