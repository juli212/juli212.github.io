Ruby Classes

February 28, 2016

A Ruby Class?

As we know, all Ruby classes are objects and each individual instance of a class is an object as well. But what exactly is the purpose of creating a new class? All classes have defined methods that are shared amongst each instance of the class. We can also designate instance variables which are available to be defined with a different value for each instance value of the method. Once defined, an instance variable represents a characteristic of a class instance and can hold a unique value.

For this topic, I will use cats as an example. Similar to defining a method, we define a class and make sure to end it. Class names usually start with a capital We can define the class called Cats below: 

        		
        		class Cats

        		end

Inside the class definition we will initialize the class and add some instance variables. In this case, when an instance of the class Cat is created, it will take three arguments: name, age, and color which will be set to instance variables. Instance variables are defined with an @ symbol in front of the variable name as seen below: 

        		
        		class Cat
        			def initialize(name, age, color)
        				@name = name
        				@age = age
        				@color =color
        			end
        		end

Similar to instance variables, instance methods can be defined inside a class and are avaiable actions for each instance of that class to take. In the class Cat, we can define the instance methods eat, pounce, and nap: 

        		
        		class Cats
        			def initialize(name, age, color)
        				@name = name
        				@age = age
        				@color =color
        			end

        			def eat(food)
        				# code to eat food
        			end

        			def pounce(object)
        				# code to pounce on object
        			end

        			def nap(minutes)
        				# code to nap for some minutes
        			end
        		end

In order for us to call the values of our instance variables for a particular instance, we also have to define methods for that. We can add the method to return an instance's value for age like so: 

        		
        		def age
        			@age
        		end

Or... The other option is to use the accessors which will replace the methods above and replace defining the instance variables in the initialization of the class. In the example below, age and color can be both read and written so they become attr_accessor 's while name is just read. This means that the value of the instance variable can be returned for any instance of Cat but it cannot be reassigned or changed. 

        		
        		attr_accessor :age, :color
        		attr_writer
        		attr_reader :name

Now that we have our Cat class defined, I will define a new instance of a cat names apollo with its three arguments. Below it, we can call the instance variable age attribute on apollo followed by the class method pouce. 

        		
        		> apollo = Cat.new("apollo", 7, "black")
        		
        		> apollo.age
        		=> 7

        		>apollo.pounce(mouse)
        		=> # Cat apollo will pounce on mouse

In this class, the instance variables age and color of an instance of the Class method can be redefined so the cat can age and you can dye its fur! The instance variable name however, cannot be changed. This is similar to a situation where you are adopting a cat and are not supposed to rename it. In addition to this, the instances of cat can all eat, ounce on things, and nap!
