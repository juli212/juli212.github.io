JavaScript vs Ruby: LOOPS

March 05, 2016

Why We Loop

Looping is a great way to make code more efficient and readable by removing repetitive lines, as long as you make sure to end the loop! Otherise the deadly infinite loop will destroy everything (or at least freak out when you're testing it...). Looping through objects is a great way to save time and space when writing code. The processes when using JavaScript and Ruby are pretty similar, though the syntax may differ slightly.

Below I will give a simple loop in JavaScript which will print out the numbers 1-5 multiplied by 10: 

        		
                //JavaScript:
                for (var i = 1; i < 6; i++) {
                    console.log(i*10);
                }

                >10
                >20
                >30
                >40
                >50

And now the same loop in Ruby: 

                
                # Ruby:
                i = 1
                while i < 6 do
                    x = i * 10
                    i = i + 1
                    puts "#{x}"
                end

                >10
                >20
                >30
                >40
                >50

In both examples above, we get the same results through the loops, even though they are constructed differently. It is very interesting to see the similarities between Ruby and JavaScript since Ruby is a server-side language while JavaScript is client-side. Despite their differences, Ruby and JavaScript can preform many of the same tasks, just using different syntax.

If you are wondering which language to use, it makes most sense to use whichever language you know better or are attempting to learn more of, though it would also make sense to take into account whether your code will be server-side or client-side.
